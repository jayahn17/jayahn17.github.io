## Caliber - Open-Source Projects at UC Berkeley

### Context
Caliber was built in ENGIN 283 (AI Startup) to solve a practical discovery problem: undergraduate project work at Berkeley was scattered, difficult to browse, and easy to miss. The goal was to reduce discovery time for students, alumni, and collaborators searching for reusable work.

### Challenge
- Consolidate 500+ projects into a consistent browsing system.
- Design fast discovery workflows through filtering, categorization, and clear presentation.
- Prepare a scalable product foundation that could later support file-management features.

### Product and Technical Approach
**Frontend and Information Architecture**
- Defined the metadata schema used for project cards, search indexing, and detail pages.
- Designed a responsive UI with a clear visual hierarchy across devices.
- Prioritized low-friction discovery through category- and technology-based filters.

**Data Management**
- Structured project ingestion and normalization so contributor submissions used consistent fields.
- Added display controls to keep technical metadata readable and comparable.
- Prepared the platform for future file-management features by separating metadata logic from presentation components.

**Validation and Communication**
- Built and shared a final pitch deck for stakeholders.
- Iterated quickly on copy, taxonomy, and information flow based on user feedback.

### Results
- Delivered a searchable interface for 500+ UC Berkeley open source projects.
- Reduced manual discovery effort through structured cards and consistent project detail pages.
- Created an extensible foundation for future workflow features, including file management.
